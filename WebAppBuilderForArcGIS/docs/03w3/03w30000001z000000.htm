Chart

The Chart widget displays one or more quantitative attributes for a feature as a graphical representation of data. It is designed to make it easy for end users to observe possible patterns and trends in quantitative attribute data.

The following table shows the descriptions of the configurable attributes.

Attribute

Description

charts

Object[]. There is no default. --Array of chart tasks. Depending on the chart mode, each task may have following attributes.

  • name: String; There is no default— The name of chart tasks displayed in the widget.
  • url: String; There is no default. -A feature layer URL for the layer you want to perform chart task.
  • filter: Object. There is no default. -The 'expr' attribute of filter takes the form of a basic SQL statement and uses it to filter features.
  • description: String. There is no default. -The description of chart task.
  • mode: String. There is no default. -The mode of chart task. The available values are feature, category, count and field. If feature is selected, chart task displays values feature by feature; If category is selected, chart task displays values by category; If count is selected, chart task displays feature counts by category; If field is selected, chart task displays attribute values as charts.
  • symbol: Object. There is no default -This is the feature symbol of chart task.
  • highLightColor: String. There is no default -The highlight color of selected features.
  • labelField: String; There is no default— The field is used as chart axis label. This attribute exists when the mode is feature.
  • valueFields: String[]. There is no default--The fields provide values to statistics. This attribute exists when mode is feature, category or field.
  • categoryField: String. There is no default--The field that is used to categorize the chart. This attribute exists when mode is category or count.
  • operation: String. There is no default.--The attribute indicates which math method the chart task uses to calculate statistics. The available values are average, sum, max and min. This attribute exists when mode is category or field.
  • webMapLayerId: String. There is no default—The id of the source layer. If webMapLayerId is set, attribute url will be ignored.
  • types: Object array. There is no default—Each element of types array is an object that contains two properties: type and display. The former is string and has valid values including column, pie, bar and line. The latter is object. See the example below for details.
  • sortOrder: Object. There is no default— It has a property named isAsc. isAsc is boolean and defines how to sort the category field value or label field value. When the value is true,ascending order applies; false, descending order applies. The attribute only exists when the display mode is feature, category or count.
